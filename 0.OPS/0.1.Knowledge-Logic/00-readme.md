# 0.1.Knowledge-Logic

## Назначение папки

Эта папка содержит **фундаментальные знания о методологии S2R** (Systems-Roles Repository) — логику организации знаний через multi-kernel структуру.

**Аудитория:** ИИ-агенты, архитекторы знаний, администраторы репозитория

**Семейство:** F0 (Мета-эпистема)

## Содержание

### Основная модель (01-03)

- **01-kernels-model.md** — Модель ядер: что такое kernel, SoI, 3×3 матрица
- **02-document-families.md** — Семейства документов F0-F9, роли разработчиков, SoTA-методы
- **03-our-systems-map.md** — Граф системных отношений для нашей экосистемы

### Эпистемология (04-06)

- **04-ontology.md** — Онтология S2R: что есть система, роль, связи (∋, ⊃, →)
- **05-glossary.md** — Глоссарий терминов S2R и предметной области
- **06-taxonomy.md** — Таксономия: классификация документов, типы, оси

### Правила и паттерны (07-09)

- **07-naming.md** — Правила именования ядер, систем, ролей, документов
- **08-anti-patterns.md** — Типичные ошибки и как их избежать
- **09-examples-library.md** — Библиотека примеров правильных структур

## Связь с другими папками 0.OPS

## Для ИИ-агентов

**При работе с репозиторием обязательно загружай:**
1. **01-kernels-model.md** — понять структуру ядер
2. **03-our-systems-map.md** — понять граф систем
3. **05-glossary.md** — понять терминологию
4. **07-naming.md** — понять правила именования

**При создании нового документа:**
1. Определи ядро (A/B/C) — какая SoI?
2. Определи систему (1/2/3) — надсистема/SoI/создания?
3. Определи роль (.1/.2/.3) — Meaning/Architecture/Operations?
4. Проверь семейство (F0-F9) и целевую аудиторию
5. Следуй стандартам из **10-standards.md**
