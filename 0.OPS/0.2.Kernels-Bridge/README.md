## Ключевые концепции

### Ядра (Kernels)

В нашем хранилище 3 ядра:

- **A.Созидатель** — SoI: Созидатель (человек)
- **B.Экосистема** — SoI: Экосистема развития (составная система)
- **C.ИТ-Платформа-Экосистемы** — SoI: ИТ-платформа (техническая подсистема)

### Типы связей

**∋ (Contains)**: Надсистема содержит SoI
- A1.Команды ∋ A2.Созидатель
- B1.Мир ∋ B2.Экосистема

**⊃ (Includes)**: Система включает подсистему
- B2.Экосистема ⊃ C2.ИТ-Платформа (как подсистему)
- B3.Система-Создания ⊃ C3.ИТ-Служба

**→ (Creates)**: Система создания создаёт SoI
- A3.Экосистема-Развития → A2.Созидатель
- B3.Система-Создания → B2.Экосистема
- C3.ИТ-Служба → C2.ИТ-Платформа

### Мосты между ядрами

**A ↔ B**: Созидатель и Экосистема
- A3.Экосистема-Развития = B2.Экосистема (разные точки зрения на одну систему)
- A2.Созидатель развивается через B2.Экосистему

**B ↔ C**: Экосистема и ИТ-Платформа
- B2.Экосистема ⊃ C2.ИТ-Платформа (платформа — подсистема экосистемы)
- B3.Система-Создания ⊃ C3.ИТ-Служба (служба — часть системы создания)

## Использование

### Для ИИ-агентов

**При создании нового документа в ядре**:
1. Загрузите [[01-value-chain.md]] — понять, как ядро вписывается в цепочку ценности
2. Загрузите [[02-kernels-relations.md]] — определить связи с другими ядрами
3. Загрузите [[03-systems-relations.md]] — проверить корректность отношений ∋⊃→

**При анализе связности хранилища**:
1. Проверьте [[03-systems-relations.md]] — все ли связи корректны
2. Проверьте [[02-kernels-relations.md]] — есть ли мосты между ядрами

### Для методологов

**При разработке нового ядра**:
1. Определите SoI нового ядра
2. Определите связи (∋⊃→) по [[03-systems-relations.md]]
3. Интегрируйте в цепочку ценности [[01-value-chain.md]]
4. Обновите матрицу [[02-kernels-relations.md]]
