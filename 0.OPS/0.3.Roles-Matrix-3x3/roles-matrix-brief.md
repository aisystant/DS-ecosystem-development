## Быстрая навигация по ролям

### FX1: Продвиженец (Надсистема × Meaning)
**В одной строке**: Переводит проблемы людей в задачи для системы

**Ключевые слова**: JTBD, проблемы, сегменты, оффер, спрос

**Типичные документы**: Карты задач (JTBD), гипотезы ценности, результаты тестов сообщений

---

### FX2: Владелец продукта (Надсистема × Architecture)
**В одной строке**: Формулирует сценарии использования системы

**Ключевые слова**: Product Discovery, сценарии, прототип, пользовательская работа

**Типичные документы**: User Story Map, сценарии использования, дорожная карта

---

### FX3: PR/GR (Надсистема × Operations)
**В одной строке**: Выстраивает отношения с внешними ролями

**Ключевые слова**: репутация, партнёры, регуляторы, медиа, доверие

**Типичные документы**: План коммуникаций, медиа-стратегия, карты ролей в надсистеме

---

### FX4: Визионер ЦС (Целевая система × Meaning)
**В одной строке**: Формирует образ целевой системы

**Ключевые слова**: функция, границы, ценность, принципы, системное видение

**Типичные документы**: Mission, Vision, границы системы, концептуальная архитектура

---

### FX5: Инженер ЦС (Целевая система × Architecture)
**В одной строке**: Проектирует архитектуру системы

**Ключевые слова**: DDD, интерфейсы, конструкция, модели, проверяемость

**Типичные документы**: Architecture Decision Records (ADR), диаграммы, модели поведения

---

### FX6: Оператор ЦС (Целевая система × Operations)
**В одной строке**: Обеспечивает эксплуатацию системы

**Ключевые слова**: SRE, SLO/SLA, инциденты, мониторинг, надёжность

**Типичные документы**: Регламенты эксплуатации, постмортемы, метрики доступности

---

### FX7: Бизнесмен (Система создания × Meaning)
**В одной строке**: Определяет стратегию и инвестиции

**Ключевые слова**: стратегия, портфель, инвестиции, риски, рост

**Типичные документы**: Бизнес-модель, стратегия, портфель проектов, финансовые прогнозы

---

### FX8: Организатор разработки (Система создания × Architecture)
**В одной строке**: Проектирует конвейер создания

**Ключевые слова**: DevOps, CI/CD, качество, автоматизация, пайплайн

**Типичные документы**: Описания процессов, CI/CD пайплайны, стандарты качества

---

### FX9: Администратор (Система создания × Operations)
**В одной строке**: Обеспечивает бесперебойную работу команды

**Ключевые слова**: люди, роли, бюджеты, ресурсы, регламенты

**Типичные документы**: Матрицы ролей, политики доступа, планы найма, финансовые отчёты

## Как использовать эту таблицу

### Для определения, куда поместить документ
1. **Определи систему**: О какой системе документ? (Надсистема / SoI / Создания)
2. **Определи роль**: Какой вопрос он отвечает? (Зачем / Как устроено / Как работает)
3. **Найди ячейку**: Пересечение системы и роли → папка X.Y

**Пример**: Документ "Бизнес-модель экосистемы"
- Система: 3 (Система создания → Экосистема)
- Роль: 1 (Meaning → Зачем, бизнес)
- Результат: A3.1.Meaning (FX7: Бизнесмен)

### Для выбора методов разработки
1. **Найди роль** в таблице SoTA-методов
2. **Выбери метод** из списка для роли
3. **Примени метод** при создании документа

**Пример**: Создаём сценарии использования → FX2 (Владелец продукта) → Product Discovery, User Story Mapping

### Для проверки полноты ядра
1. **Проверь наличие всех 9 папок**: X1.1, X1.2, X1.3, X2.1, X2.2, X2.3, X3.1, X3.2, X3.3
2. **Проверь, что каждая папка** содержит документы соответствующей роли
3. **Проверь применение методов**: используются ли SoTA-методы для роли
