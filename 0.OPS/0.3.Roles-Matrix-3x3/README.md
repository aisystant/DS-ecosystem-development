# 0.3.Roles-Matrix-3x3

> **Для ИИ-агентов:** Этот раздел объясняет фундаментальную структуру S2R — матрицу 3×3 (3 системы × 3 роли = 9 папок). Используйте для понимания логики организации каждого ядра.

## О разделе

**0.3.Roles-Matrix-3x3** объясняет математическую основу S2R методологии: почему каждое ядро состоит из **9 папок** и какие роли отвечают за какие области.

### Ключевая идея

Каждое ядро (Kernel) в S2R — это полная матрица 3×3:

```
3 СИСТЕМЫ × 3 РОЛИ = 9 ПАПОК

Системы:                 Роли:
├─ X1. Надсистема        ├─ X.1. Meaning (Смыслы)
├─ X2. Целевая система   ├─ X.2. Architecture (Архитектура)
└─ X3. Система создания  └─ X.3. Operations (Операции)
```

**Например**, для ядра A (Созидатель):
- A1.Мир (надсистема) → A1.1.Meaning, A1.2.Architecture, A1.3.Operations
- A2.Созидатель (SoI) → A2.1.Meaning, A2.2.Architecture, A2.3.Operations
- A3.Экосистема (создания) → A3.1.Meaning, A3.2.Architecture, A3.3.Operations

**Итого**: 9 папок = полная модель системы

## Зачем нужна матрица 3×3?

### 1. Полнота охвата
Матрица гарантирует, что мы рассматриваем систему со **всех важных углов**:
- 3 уровня систем (надсистема, SoI, создания)
- 3 способа взаимодействия (смыслы, архитектура, операции)

### 2. Разделение ответственности
Каждая **проектная роль** имеет чёткую зону ответственности:
- **Предприниматель** (X.1) — отвечает за "Зачем?" (Meaning)
- **Инженер** (X.2) — отвечает за "Как устроено?" (Architecture)
- **Менеджер** (X.3) — отвечает за "Как работает?" (Operations)

### 3. Связь с семействами документов
Матрица 3×3 напрямую связана с семействами документов F1-F9:

| Система ↓ / Роль → | Meaning (1) | Architecture (2) | Operations (3) |
|---|---|---|---|
| **Надсистема (1)** | F1 | F2 | F3 |
| **SoI (2)** | F4 | F5 | F6 |
| **Создания (3)** | F7 | F8 | F9 |

Подробнее: [0.1/02-document-families.md](../0.1.Knowledge-Logic/02-document-families.md)

### 4. Воспроизводимость структуры
Одна и та же матрица 3×3 применяется к **любому ядру**:
- **A.Созидатель** = 9 папок (A1.1 - A3.3)
- **B.Экосистема-развития** = 9 папок (B1.1 - B3.3)
- **C.ИТ-платформа-Экосистемы** = 9 папок (C1.1 - C3.3)

Это делает структуру **предсказуемой** и **масштабируемой**.

## Три роли в деталях

### .1. Meaning (Смыслы)
**Вопрос**: Зачем? Какова ценность?

**Фокус**: Миссия, видение, ценностное предложение, бизнес-модель

**Примеры документов**:
- Mission Statement (зачем система существует)
- Value Proposition (какую ценность приносит)
- Business Model (как зарабатывает)

**Роли-разработчики**: Визионер, Бизнесмен, Продвиженец

### .2. Architecture (Архитектура)
**Вопрос**: Как устроено?

**Фокус**: Структура, компоненты, модель, интерфейсы

**Примеры документов**:
- System Architecture (как устроена система)
- Component Diagram (из чего состоит)
- Data Model (как организованы данные)

**Роли-разработчики**: Инженер, Архитектор, Владелец продукта

### .3. Operations (Операции)
**Вопрос**: Как работает?

**Фокус**: Процессы, практики, эксплуатация, ритуалы

**Примеры документов**:
- Operating Procedures (как работает система)
- User Journey (как пользователь проходит путь)
- Development Process (как разрабатывается)

**Роли-разработчики**: Оператор, Администратор, Менеджер

## Три системы в деталях

### 1.X. Надсистема (Supersystem)
**Отношение**: ∋ (содержит SoI)

**Фокус**: Внешний мир, в котором живёт система

**Примеры**:
- Для Созидателя → Мир (потенциальные участники, рынок)
- Для Экосистемы → Образовательный рынок
- Для ИТ-Платформы → Технологический ландшафт

**Роли**: FX1 (Продвиженец), FX2 (Владелец продукта), FX3 (PR/GR)

### 2.X. Целевая система (System of Interest)
**Отношение**: SoI (фокус ядра)

**Фокус**: Сама система, её границы, функция, устройство

**Примеры**:
- Для ядра A → Созидатель (человек, создающий ценность)
- Для ядра B → Экосистема развития (платформа для созидателей)
- Для ядра C → ИТ-Платформа (софт и инфраструктура)

**Роли**: FX4 (Визионер ЦС), FX5 (Инженер ЦС), FX6 (Оператор ЦС)

### 3.X. Система создания (Creation System)
**Отношение**: → (создаёт SoI)

**Фокус**: Как мы создаём целевую систему

**Примеры**:
- Для Созидателя → Экосистема развития (кто обучает)
- Для Экосистемы → Команда и службы (кто строит)
- Для ИТ-Платформы → DevOps и команда разработки

**Роли**: FX7 (Бизнесмен), FX8 (Организатор разработки), FX9 (Администратор)

## Практическое применение

### Для ИИ-агентов
**Используйте матрицу 3×3 для**:
1. **Определения, куда поместить документ** — какая роль + какая система
2. **Выбора методов разработки** — какие SoTA-методы применять
3. **Проверки полноты ядра** — есть ли все 9 папок
4. **Понимания целевой аудитории** — для кого пишется документ

### Для разработчиков
**Используйте матрицу 3×3 для**:
1. **Навигации по репозиторию** — понимание структуры любого ядра
2. **Определения зоны ответственности** — кто за что отвечает
3. **Создания новых ядер** — воспроизводимый шаблон 9 папок
4. **Анализа полноты** — проверка, что все роли покрыты
