## «Мосты» — формальные связки между смыслами, данными и день‑за‑днём

1. **Смысл → Слот**: миссия/цели каскадируются через эпики/спринты в дневные слоты; в каждой учебной единице поле «где применю за 7–14 дней».    
2. **Проблема → Гипотеза → Интервенция → Метрики**: формат HYP/INT со способами опровержения (перенос, артефакт, рецензия).    
3. **ConOps → REQ → Архитектура → События → Данные → Дашборды**: сценарии ролей порождают требования, компоненты, события и кубы. 
4. **Роль/Ступень → Пороги → Доступы → Выплаты/Квалификации**: прозрачные критерии → автоматические действия.    
5. **Руководство → Учебная единица → Задача → Артефакт → Peer‑review → Токены** (цепочка ценности).    
6. **Global‑Core ↔ Local‑Edge**: общие реестры ↔ локальные адаптеры/комплаенс.
