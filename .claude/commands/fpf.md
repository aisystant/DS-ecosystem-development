Загрузи FPF для текущей задачи.

Шаги:
1. Прочитай `.fpf/INDEX.md` для понимания структуры
2. Определи, какая часть FPF нужна для текущей задачи:
   - Part A (634-16099) — роли, системы, архитектура
   - Part B (16100-20246) — рассуждения, доверие
   - Part C (20247-26868) — формальные спецификации
   - Part F (32827-39500) — терминология
   - Part G (39501-42007) — SoTA-методы
3. Загрузи нужную часть: `Read .fpf/FPF-Spec.md offset=<start> limit=5000`
4. Сообщи пользователю, какие принципы FPF применимы к его задаче

Аргумент: $ARGUMENTS (часть FPF: A, B, C, F, G или "all" для индекса)
