## ОБЯЗАТЕЛЬНО: Архитектура мышления FPF

**ВСЕГДА применяй First Principles Framework (FPF)** Анатолия Левенчука при работе с этим репозиторием.

### Когда загружать FPF

**Загрузи соответствующую часть FPF-Spec.md ПЕРЕД:**
- Анализом ролей, систем, архитектуры → Part A (строки 634-16099)
- Рассуждениями, оценкой доверия → Part B (строки 16100-20246)
- Созданием спецификаций → Part C (строки 20247-26868)
- Работой с терминологией → Part F (строки 32827-39500)
- Применением SoTA-методов → Part G (строки 39501-42007)

### Как применять ADI-цикл

При любой нетривиальной задаче **ОБЯЗАТЕЛЬНО** проходи цикл:
1. **Abduction** — сгенерируй 2-3 гипотезы (НЕ останавливайся на первой!)
2. **Deduction** — проверь логику каждой гипотезы
3. **Induction** — собери evidence из кода/документов
4. **Audit** — проверь на когнитивные искажения
5. **Decide** — выбери лучшую гипотезу, задокументируй rationale

### Ключевые принципы (применяй всегда):

1. **Holonic (A.1):** Всё — холон (одновременно целое и часть)
2. **Bounded Context (A.1.1):** Значение локально в контексте
3. **Strict Distinction (A.7):** Роль ≠ Метод ≠ Работа; Документ ≠ Система
4. **Trust Calculus (B.3):** Доверие = ⟨F, G, R⟩
5. **ADI Cycle (B.5):** Abduction → Deduction → Induction
